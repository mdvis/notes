3. **Code review 时的“隐形规范”**
    
### 怎么用这份结构（实操建议）
- 🧱 **起新项目**：  
    直接复制整个骨架，先删掉用不到的目录（而不是一开始就全堆在 flake.nix 里）
    
- 🔍 **判断一个 Flake 是否“成熟”**：  
    对照这几个问题：
    
    - flake.nix 是不是主要在 import？
        
    - 构建 / 运行 / 开发有没有清晰分层？
        
    - 有没有把系统配置和模块混在一起？
        
- 🧠 **进阶演化路线**：
    
    - 个人项目：`pkgs + devShells`
        
    - 团队项目：+ `checks + lib`
        
    - 平台/基础设施：+ `modules + systems`
        
    - 社区输出：+ `templates`
