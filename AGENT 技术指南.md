## 一、 构建 AI Agent 的 11 大 Prompt 核心技术

构建智能 Agent 的核心目标是提升其逻辑推理与自我修正能力 1。

|**编号**|**技术名称**|**核心思想**|**示例场景**|
|---|---|---|---|
|1|**Self-Consistency**|让模型多次推理，取最一致的答案 2|数学题、逻辑推理 3|
|2|**Tree of Thoughts (ToT)**|树状探索多个推理路径，评估并选择最佳路径 4|复杂规划、策略游戏 5|
|3|**Chain of Thought (CoT)**|逐步推理，显式展示思考链（“Let's think step by step”） 6|所有需要推理的任务 7|
|4|**ReAct**|交替进行“思考”与“行动”（工具调用） 8|工具调用、交互任务 9|
|5|**Reflexion**|让 Agent 自我反思错误并迭代优化 10|试错学习、任务优化 11|
|6|**Plan-and-Execute**|先制定完整计划，再按步骤执行 12|长任务分解 13|
|7|**Multi-Agent Debate**|多个角色辩论，通过观点平衡得出结论 14|决策冲突、复杂评估 15|
|8|**Self-Ask with Search**|主动提出子问题并通过搜索补充知识 16|知识密集型问答 17|
|9|**Graph of Thoughts (GOT)**|非线性组合多个思考模块 18|创意生成、知识融合 19|
|10|**Skeleton-of-Thought**|先给出答案骨架，再并行填充内容 20|长文本生成加速 21|
|11|**自动 Prompt 优化**|利用 LLM 自动改进原始提示词以获得更好结果 22|Prompt 工程自动化 23|

> **专家建议：** 推荐组合使用 **CoT + ReAct + Reflexion** 来构建具备自省能力的智能 Agent 24。

## 二、 Google 官方 Prompt 工程框架 (2024版)

Google 将 Prompt 工程视为“LLM 的编程” 25，并提出了设计与评估的标准。

1. 设计三原则 26

- **清晰性 (Clarity)：** 使用具体动词，避免模糊表达 27。
    
- **明确性 (Specificity)：** 指定输出格式（如 JSON）或明确数量（如列出3点） 28。
    
- **上下文 (Context)：** 提供必要的角色设定、任务背景及约束条件 29。
    

2. 评估与迭代 30

- **科学评估：** 使用 BLEU、ROUGE 等指标或人工评价 (Human Eval) 来衡量输出质量 31。
    
- **对比测试：** 对不同版本的 Prompt 进行 A/B 测试 32。
    
- **版本控制：** 建立 Prompt 的版本控制系统以记录改进过程 33。

## 三、 IDEAS 框架与结构化模块设计

《Prompt Engineering Playbook》提供了一套实用的操作手册 34。

1. IDEAS 结构化方法论 35

- **I - Intent (意图)：** 明确你的目标 36。
    
- **D - Design (设计)：** 构建 Prompt 结构 37。
    
- **E - Examples (示例)：** 提供 Few-Shot 示例 38。
    
- **A - Adapt (调整)：** 根据反馈进行修正 39。
    
- **S - Scale (扩展)：** 将成功的模式推广应用 40。
    

2. 模块化 Prompt 模板 41

一个成熟的 Prompt 应包含以下五个标准模块：

1. **角色 (Role)：** 设定专业身份，如“你是一位资深数据分析师” 42。
    
2. **任务 (Task)：** 定义具体动作，如“用3个要点总结核心洞察” 43。
    
3. **格式 (Format)：** 规定输出形式，如“以 JSON 格式响应” 44。
    
4. **约束 (Constraints)：** 设定限制，如“字数控制在100字以内，不使用术语” 45。
    
5. **示例 (Examples)：** 提供少样本 (Few-Shot) 学习示例 46。
