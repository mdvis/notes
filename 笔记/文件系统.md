- **NTFS (New Technology File System)**  
 支持文件权限、加密、压缩及日志功能，具备高安全性与稳定性，适用于现代Windows系统的硬盘分区。其最大支持2TB的分区，且通过日志功能确保数据一致性。  
- **FAT32/exFAT**  
 FAT32兼容性极强，广泛用于U盘和移动设备，但文件大小限制为4GB；exFAT优化了大文件支持，适合大容量闪存设备。
- **ext4 (Fourth Extended File System)**  
 主流的Linux文件系统，支持日志功能、延迟分配以减少碎片，适合高性能服务器环境。最大支持1EB的单一文件系统容量。  
- **XFS/Btrfs**  
 XFS适用于大文件处理和高吞吐场景（如视频编辑）；Btrfs支持快照、动态卷管理和数据校验，适合需要高级功能的场景。
- **APFS (Apple File System)**  
 专为SSD优化，支持快速克隆、加密和快照功能，提升存储效率与数据安全性，是macOS和iOS的默认文件系统。
- **exFAT**  
Microsoft与SD协会联合开发，支持大文件和跨平台（Windows/macOS/Linux），适合外部存储设备。  
- **NTFS-3G**  
通过用户态驱动在非Windows系统中读写NTFS分区，解决跨平台兼容问题。
 
**企业级存储**  
- **ZFS (Zettabyte File System)**  
 集成文件系统与卷管理，支持数据校验、自动修复、快照和压缩，适用于高可用性需求的企业级存储，如数据中心和云存储。  
- **Btrfs (B-Tree File System)**  
 类似ZFS，支持动态卷扩展、数据镜像和快照，常用于需要灵活管理的存储环境。
 
**分布式存储**  
- **Ceph**  
 提供对象、块和文件存储的统一接口，适用于大规模分布式存储集群，支持高扩展性和容错性。  
- **JuiceFS**  
 基于FUSE构建，结合对象存储与分布式缓存，支持POSIX接口，适合云原生环境的大数据与AI训练场景。

**AI训练与推理**  
- **3FS (Fire-Flyer File System)**  
 DeepSeek开发的高性能并行文件系统，支持RDMA网络和SSD优化，单节点峰值带宽达40 GiB/s，适用于大规模模型训练和推理的IO密集型任务。  
- **Lustre**  
 常用于超级计算中心，支持PB级数据并行访问，适合HPC（高性能计算）场景。
 
**实时数据处理**  
- **XFS**  
 高吞吐量和低延迟特性使其适合实时日志处理和大数据分析。
 
**嵌入式系统**  
- **FAT32**  
 因结构简单、兼容性强，广泛用于嵌入式设备和小型存储介质（如SD卡）。  
- **YAFFS2 (Yet Another Flash File System)**  
 专为NAND闪存设计，支持磨损均衡，常见于嵌入式Linux设备。
 
**移动端优化**  
- **F2FS (Flash-Friendly File System)**  
 针对闪存特性优化，减少写入放大，提升SSD和移动设备寿命，被Android和部分Linux发行版采用。
 
**网络文件系统 (NFS)**  
- **NFS (Network File System)**  
 允许跨网络挂载远程目录，实现多机共享存储，常用于企业内网和云计算环境。  
- **GlusterFS**  
 分布式文件系统，支持横向扩展，适用于需要高可用性和负载均衡的场景。

**虚拟化与容器**  
- **OverlayFS**  
 联合挂载技术，支持容器镜像分层存储，是Docker等容器引擎的默认文件系统。
 
**加密与安全**  
- **eCryptfs**  
 基于Linux的加密文件系统，支持目录级透明加密，保护敏感数据。  
- **BitLocker (NTFS集成)**  
 Windows的磁盘加密功能，与NTFS深度集成，提供全盘加密。

**只读与归档存储**  
- **ISO 9660**  
 标准CD/DVD文件系统，支持长文件名扩展（如Rock Ridge和Joliet）。  
- **ZFS归档模式**  
 通过快照和压缩实现长期数据归档，确保数据完整性。
