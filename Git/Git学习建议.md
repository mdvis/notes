# Git 命令学习建议

作为一名软件开发人员，Git 命令遵循 "二八定律"：20% 的命令覆盖了 80% 的日常工作场景。

基于当前目录结构，以下是为你整理的**优先级清单**：

## 🟢 第一梯队：核心高频 (必须熟练掌握)
这些是你每天开发、提交代码、同步进度必须用到的命令。

*   **基础操作/**
    *   `init.md` (初始化项目)
    *   `clone.md` (下载项目)
    *   `status.md` (查看当前状态，**每天用无数次**)
    *   `diff.md` (查看改了什么，提交前必看)
    *   `log.md` (查看提交历史)
*   **分支管理/**
    *   `branch.md` (新建/查看分支)
    *   `checkout.md` (切换分支)
    *   `merge.md` (合并分支)
*   **远程操作/**
    *   `remote.md` (管理远程仓库)
    *   `fetch.md` (获取远程更新，但不合并)
    *   `pull.md` (拉取并合并，最常用)
    *   `push.md` (推送代码)
*   **配置与其他/**
    *   `config.md` (配置用户名、邮箱、别名)

---

## 🟡 第二梯队：进阶与救火 (建议掌握)
掌握这些命令能让你在团队协作中处理冲突、修复错误，从“初级工”进阶为“熟练工”。

*   **分支管理/**
    *   `rebase.md` (变基：保持提交历史整洁的神器，团队开发中很重要)
    *   `cherry-pick.md` (挑拣提交：把特定提交应用到当前分支，修复 Bug 时很常用)
*   **提交与历史/**
    *   `reset.md` (重置：撤销提交，**救命神技**)
    *   `revert.md` (回滚：产生一个新的提交来撤销之前的修改，安全的回退方式)
*   **文件管理/**
    *   `restore.md` (恢复文件：丢弃工作区的修改)
    *   `clean.md` (清理未追踪文件：快速清理编译垃圾或临时文件)
*   **高级功能/**
    *   `stash.md` (暂存：开发到一半需要切分支修 Bug 时非常有用)
*   **标签与版本/**
    *   `tag.md` (打标签：发布版本时使用)

---

## 🔵 第三梯队：特定场景 (按需查阅)
这些命令通常用于特定需求（如 Debug、复杂的项目结构），需要用到时再去查阅文档即可。

*   **提交与历史/**
    *   `bisect.md` (二分查找：**Debug 神器**，用于在历史记录中定位是哪个提交引入了 Bug)
*   **高级功能/**
    *   `worktree.md` (工作树：允许你同时检出多个分支到不同目录，并行开发利器)
    *   `submodule.md` / `subtree.md` (子模块/子树：只有当你的项目引用了其他 Git 仓库时才需要)
*   **维护与清理/**
    *   `archive.md` (归档：将代码打包成 zip/tar，不带 .git 目录)
*   **配置与其他/**
    *   `grep.md` (搜索代码：命令行搜索)

---

## ⚪ 第四梯队：底层与内部原理 (无需刻意掌握)
这些通常被称为 **Plumbing (管道/底层) 命令**，它们是给脚本或 Git 内部使用的，日常开发几乎不会直接手敲。

*   **📂 底层命令/ (整个文件夹均可忽略)**
    *   `cat-file.md`, `object.md`, `rev-parse.md`, `update-index.md` 等。
*   **文件管理/**
    *   `ls-files.md` (查看索引区文件列表)
*   **提交与历史/**
    *   `cherry.md` (查看未推送到上游的提交)
*   **维护与清理/**
    *   `filter-branch.md` (重写历史，极度危险且慢)
*   **标签与版本/**
    *   `describe.md` (生成可读的版本号)

## 💡 总结建议
1.  **重点攻克** 🟢 第一梯队。
2.  **理解原理** 🟡 第二梯队，特别是 `reset` 和 `rebase`。
3.  **📂 底层命令/** 除非深入研究原理，否则可跳过。
