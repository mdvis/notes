查找哪一次代码提交引入了错误。
## 两分法
两分法就是将代码历史一分为二，确定问题出在前半部分，还是后半部分，不断执行这个过程，直到范围缩小到 某一次代码提交。
## 原理
bisect的原理很简单，就是将代码提交的历史，按照两分法不断缩小定位。
## 过程
```bash
$ git bisect start [终][起点] # 启动查错
# "终点"是最近的提交
# "起点"是更久以前的提交
# 它们之间就是查错范围

$ git bisect start HEAD 4d83cf
# 代码库就会切换到这段范围正当中的那一次提交

$ git bisect good
# 标记为无问题，说明此次提交的后续提交引入的问题（二分的后半部分）
# 切换到后半段中点那次提交

$ git bisect bad
# 标记为有问题，说明此次提交前面的提交引入的问题(第二次二分的前半段)
# 切换到问题段的中点

# 不断重复上面的过程直至找到某次提交

$ git bisect reset
# 退出查错，回到最近一次的代码提交
# (第100至第51次提交) ## git切换为后半段的中点，第76次提交
$ b47892 is the first bad commit
# 接下来，不断重复这个过程，直到成功找到出问题的那一次提交为止。
```
