这类编码将数据结构或对象转换为可存储或传输的格式。

#### 23. JSON
- **原理**： 基于JavaScript对象的轻量级文本数据格式，使用键值对和数组。
- **用途**： Web API数据交换、配置文件。
- **特点**： 易读、解析简单，支持多种数据类型。

#### 24. XML
- **原理**： 使用标签定义数据的标记语言，可自定义结构。
- **用途**： 文档存储、Web Service（SOAP）、配置文件。
- **特点**： 结构严谨、可扩展，但冗余大。

#### 25. YAML
- **原理**： 使用缩进和符号表示数据的格式，强调可读性。
- **用途**： 配置文件（如Docker Compose、Kubernetes）。
- **特点**： 人类可读性好，支持复杂数据结构。

#### 26. Protobuf
- **原理**： Google开发的二进制序列化格式，需预定义模式（.proto文件）。
- **用途**： 高性能RPC通信、数据存储。
- **特点**： 体积小、解析快，但需预编译。

#### 27. Avro
- **原理**： Apache开发的二进制序列化格式，模式存储在JSON文件中。
- **用途**： 大数据系统（如Hadoop、Kafka）。
- **特点**： 支持模式演化，适合大数据场景。

#### 28. MessagePack
- **原理**： 二进制JSON格式，类型信息与数据一起存储。
- **用途**： 网络通信、缓存数据序列化。
- **特点**： 比JSON更紧凑，解析速度快。
