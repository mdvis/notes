## 七层模型（OSI模型）
**由来**： 国际标准化组织（ISO）于1984年提出的理论框架，旨在统一网络通信标准，尽管理论完备但实际应用较少。
### 各层详解
1. **物理层（Layer 1）**
	- **作用**：传输原始比特流，定义物理媒介特性（如电缆类型、电压）。
	- **协议**：RS-232、光纤、IEEE 802.3（以太网物理层）。
2. **数据链路层（Layer 2）**
	- **作用**：相邻节点间传输数据帧，提供错误检测与MAC寻址。
	- **协议**：Ethernet、Wi-Fi、PPP、VLAN（虚拟局域网）。
3. **网络层（Layer 3）**
	- **作用**：跨网络路由与逻辑寻址，处理拥塞控制。
	- **协议**：IP、ICMP、OSPF（路由协议）。
4. **传输层（Layer 4）**
	- **作用**：与TCP/IP传输层相同，提供端到端服务。
	- **协议**：TCP、UDP。
5. **会话层（Layer 5）**
	- **作用**：管理会话连接（建立、同步、终止）。
	- **协议**：NetBIOS、PPTP（虚拟专用网）。
6. **表示层（Layer 6）**
	- **作用**：数据格式转换、加密/解密、压缩/解压缩。
	- **协议**：SSL/TLS（加密）、JPEG（图像编码）、ASCII/Unicode。
7. **应用层（Layer 7）**
	- **作用**：直接为用户应用提供接口（如文件传输、邮件）。
	- **协议**：HTTP、FTP、SMTP、DNS。
## 模型对比与核心差异

| **特性**   | **四层模型（TCP/IP）**      | **七层模型（OSI）**      |
| -------- | --------------------- | ------------------ |
| **设计理念** | 实用驱动，简化分层（如合并会话层）     | 理论先行，严格分层（7层清晰划分）  |
| **层级合并** | 应用层整合OSI上三层（会话、表示、应用） | 独立分层（会话层、表示层）      |
| **协议重点** | 强调IP、TCP/IP协议族        | 覆盖更多理论协议（如RPC、NFS） |
| **适用场景** | 互联网实际通信（Web、邮件）       | 教学、协议分析（如网络排错）     |
## 关键协议与技术
1. **TCP三次握手**：客户端发送`SYN` → 服务器`SYN-ACK` → 客户端`ACK`，建立可靠连接。
2. **DNS解析流程**：域名 → 本地缓存 → 递归查询根服务器 → 迭代查询权威服务器 → 返回IP地址。
3. **HTTP与HTTPS**：HTTP明文传输，HTTPS通过SSL/TLS加密（表示层+应用层协作）。
4. **NAT（网络地址转换）**：网络层技术，允许多设备共享公网IP。
## 总结
- **TCP/IP模型**是互联网的“事实标准”，以高效实用为核心，广泛用于Web、电子邮件等场景。
- **OSI模型**是理论框架，通过分层思想帮助理解网络协议，但实际应用较少。
- **协议对应关系**：四层模型的应用层包含OSI的会话层、表示层功能（如SSH管理会话、SSL加密数据）。 通过对比学习，可更深入理解网络通信的本质，为网络设计、故障排查提供理论支持。
