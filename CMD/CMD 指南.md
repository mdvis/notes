# Linux 命令学习指南 (软件开发人员版)

这份指南将命令按照软件开发日常工作的相关性进行了分级，帮助你更有针对性地掌握核心技能。

---

## 🟢 第一梯队：必须掌握 (生存与日常开发)
**这些命令是日常工作的基石，建议达到“肌肉记忆”程度。**

### 1. 开发与容器
- `git`: 版本控制（现代开发必不可少）。
- `docker`: 容器化部署与本地环境搭建。
- `make`: 编译工程与执行 Makefile 自动化任务。

### 2. 基础导航与文件操作
- `ls`, `cd`, `pwd`: 目录切换与查看。
- `cp`, `mv`, `rm`: 文件的增、删、改、移。
- `mkdir`, `touch`: 创建目录和空文件。
- `ln`: 创建符号链接（在处理路径或部署时常用）。
- `cat`: 快速查看短文件内容。

### 3. 日志查看与文本搜索
- `tail`: 特别是 `tail -f`，用于实时监控程序运行日志。
- `grep`: 在海量日志或代码中搜索关键字。
- `less`: 分页查看长日志，比 `more` 更强大（支持前后翻页、搜索）。

### 4. 网络调试
- `curl`: 调试 REST API、下载文件、验证网络联通性的神器。
- `ping`: 测试基础网络连通性。
- `ifconfig` / `ip`: 查看本机 IP 地址及网卡信息（容器内常用）。

### 5. 权限管理
- `chmod`: 修改文件执行/读写权限（如 `chmod +x script.sh`）。
- `chown`: 更改文件所属用户/组（解决挂载卷权限问题）。
- `sudo`: 提权执行命令。

### 6. 进程管理与端口排查
- `ps`: 查看进程运行状态（常配合 `grep` 使用）。
- `kill`: 终止进程。
- `lsof`: **极重要**，排查“端口被谁占用” (如 `lsof -i :8080`)。

## 🟡 第二梯队：建议掌握 (提升效率与脚本编写)
**掌握这些命令能让你从“新手”进阶为“高效开发者”。**

### 1. 文本流处理 (脚本利器)
- `awk`: 基础的列提取与统计（如打印进程 PID）。
- `sed`: 基础的文本替换与流编辑。
- `xargs`: 将前一个命令的输出作为后一个命令的参数。
- `sort`, `uniq`, `wc`: 统计行数、去重分析流量、排序。
- `diff`: 对比两个配置文件或代码文件的差异。

### 2. 高级查找与归档
- `find`: 根据名称、大小、修改时间精确查找文件。
- `tar`: Linux 下最通用的打包解压缩工具 (`-zcvf`, `-zxvf`)。
- `zip`, `unzip`: 处理常见的 ZIP 压缩包。

### 3. 环境配置
- `export`, `env`: 管理环境变量（如 `PATH`, `JAVA_HOME`）。
- `alias`: 设置命令别名，简化长命令。
- `history`: 快速找回之前输入过的命令。
- `source`: 让修改后的 `.bashrc` 或配置文件立即生效。

### 4. 系统监控
- `top`: 实时查看 CPU 和内存占用高的进程。
- `free`: 查看系统剩余内存。
- `df`, `du`: 查看磁盘空间占用情况（排查磁盘满报错）。
- `systemctl`: 管理系统服务的启动、停止、重启（如 `nginx`, `mysql`）。

### 5. 安全相关
- `openssl`: 证书查看、生成与格式转换。
- `md5sum`, `sha256sum`: 验证文件完整性。

## ⚪ 第三梯队：无需刻意掌握 (查文档即可)
**这些命令通常属于专业运维 (SRE) 或底层系统开发范畴。**

- **图形界面相关**: `xrandr`, `xinit`, `xxd`（除非做桌面应用）。
- **底层磁盘维护**: `grub`, `mkfs`, `fdisk`, `parted`, `mount`, `chroot`, `e2fsck`（云服务器通常已配置好）。
- **复杂网络路由**: `iptables`, `route`, `arp`, `mtr`（通常由安全组或基础架构团队管理）。
- **底层用户管理**: `useradd`, `usermod`, `groupadd`（初始化环境后极少使用）。
- **冷门文本处理**: `csplit`, `join`, `fmt`, `fold`（现代编程语言处理文本更方便）。

---

## 💡 学习建议
1. **场景导向**：不要死记硬背参数，记住“我要看端口占用该用哪个命令”比记住 `lsof` 的所有 flag 更重要。
2. **善用工具**：遇到不确定的参数，首选 `command --help` 或直接询问 AI。
3. **实践第一**：在 Docker 或远程服务器上多操作，肌肉记忆比纸上谈兵快得多。
