# 12 周强化计划（每周主题 + 具体任务）

> 建议节奏：每周 6 天学习 / 1 天复盘与项目推进。每天 6–8 小时学习（理论 + 实操）。

## 第1周 — 区块链与密码学基础（宏观与数学直观）

**目标**：理解区块链基本原理与加密基础

- 读：比特币白皮书（概要理解）与区块链工作原理概述（中文/英文博客或书）
    
- 学：哈希函数（SHA256）、公私钥、非对称签名（ECDSA 概念）、对称加密差别
    
- 实操：用现成库生成密钥对、签名与验签（Python 或 Node.js）
    
- 输出：写一份 2 页笔记，总结哈希、签名、Merkle Tree 概念及用途
    

## 第2周 — 分布式系统与共识（P2P、拜占庭容错）

**目标**：理解共识协议与网络层基本问题

- 学：PoW、PoS、PBFT、委托机制、最终性概念
    
- 实操：观看/阅读共识协议动画或示例；画图比较 PoW vs PoS
    
- 输出：写 FAQ：为什么需要共识？双花问题如何解决？
    

## 第3周 — 比特币与 UTXO 模型（交易流程）

**目标**：理解比特币交易结构、UTXO、脚本（轻量）

- 学：交易输入输出、交易费用、简要了解 Script（比特币脚本）
    
- 实操：使用 Bitcoin Core/比特币轻钱包或模拟工具构造交易（或用库演示）
    
- 输出：实现一个简单示例：构造、签名并广播（或模拟）交易
    

## 第4周 — 以太坊 & 账号模型（智能合约入门）

**目标**：掌握以太坊账户模型、交易、Gas、EVM 基础

- 学：交易生命周期、Gas 计费、EVM 栈/存储/内存简介
    
- 实操：安装 Node.js、Hardhat、Metamask；运行本地测试链（Hardhat node）
    
- 输出：部署第一个 “HelloWorld” 合约
    

## 第5周 — Solidity 基础（语法与模式）

**目标**：学会写可编译的 Solidity 合约并做单元测试

- 学：Solidity 基本语法：变量、函数、事件、modifier、错误处理、映射、数组、继承
    
- 实操：写并测试一个 ERC20 最小实现（或使用 OpenZeppelin）
    
- 输出：提交一份包含合约 + 测试的 Git 仓库
    

（**示例 ERC20 简化合约**放在后面可直接复制、编译）

## 第6周 — 智能合约开发工具链与测试

**目标**：掌握完整开发流程、工具与安全常识

- 学：Hardhat/Foundry/Truffle、OpenZeppelin、Ethers.js / Web3.js、合约测试框架（Mocha/Chai）
    
- 实操：写脚本（deploy scripts）、编写多场景单元测试（正常/异常/攻击场景）
    
- 输出：完成合约 CI 流程（本地）
    

## 第7周 — 安全实践与常见漏洞（极重要）

**目标**：熟悉常见漏洞与防御（重入、整数溢出、检查-效果-交互等）

- 学：常见漏洞列表（重入、前运行、授权问题、时间依赖、随机数安全）
    
- 实操：用测试演示典型漏洞（例如一个受攻击的简单借贷合约），并修补
    
- 输出：写一份安全检查清单（Pre-audit checklist）
    

## 第8周 — 去中心化应用（dApp）前端与后端集成

**目标**：把智能合约连接到前端，了解用户体验要点

- 学：Wallet 交互（Metamask）、gas 提示、签名请求、链上/链下数据管理（The Graph 基础）
    
- 实操：用 React + Ethers.js 做一个简单前端，能 mint 或 transfer token
    
- 输出：部署前端到 Vercel/Netlify（或本地演示）
    

## 第9周 — DeFi 基础与常见协议（交易、借贷、AMM）

**目标**：理解 AMM（Uniswap）、订单簿、借贷模型（Compound/Aave）

- 学：常见公式（x*y=k）、闪电贷原理、抵押清算流程、利率模型基础
    
- 实操：复现简化版 AMM 或闪电贷攻击演示（测试链上）
    
- 输出：写一篇技术博客解释 AMM 工作原理并给出例子
    

## 第10周 — Layer-2、可扩展性与跨链

**目标**：掌握 Rollups（ optimistic / zk-rollup）、侧链、桥的风险与原理

- 学：状态压缩、汇总交易（batching）、数据可用性问题、跨链桥攻击案例学习
    
- 实操：搭建/理解一个简化 optimistic rollup 流程（概念实现或阅读示例）
    
- 输出：做一张比较表：各扩容方案优缺点及适用场景
    

## 第11周 — 隐私、治理、代币经济学（Tokenomics）

**目标**：了解隐私技术（zk-SNARKs/zk-STARKs）、DAO 与代币设计要点

- 学：零知识证明概念、治理投票机制、通胀模型、激励/惩罚设计
    
- 实操：尝试使用已开源的 zk 库或阅读入门材料；用 Snapshot/Aragon 浏览 DAO 案例
    
- 输出：设计一个简单代币经济模型（白纸）
    

## 第12周 — 综合项目 + 面试准备 + 复盘

**目标**：完成至少一个端到端项目并准备面试材料

- 项目建议（任选其一并完成部署/文档/测试）：
    
    - ERC20 + 前端钱包交互 + 转账/空投脚本
        
    - 简化 DEX（AMM） + 前端 + 测试
        
    - NFT 市场（mint/list/buy）+ IPFS 上传（或模拟）
        
    - 借贷原型（带抵押/清算）+ 安全测试
        
- 面试准备：刷常见面试题（Solidity 问题、系统设计、Gas 优化、安全场景）
    
- 输出：GitHub 项目 + 项目说明文档 + 3–5 个技术博客/演示视频

**常用命令示例（Hardhat）**

```bash
# 初始化
npm init -y
npm install --save-dev hardhat
npx hardhat            # create a JS project
# 编译
npx hardhat compile
# 本地节点
npx hardhat node
# 部署到本地或测试网
npx hardhat run scripts/deploy.js --network localhost
# 运行测试
npx hardhat test
```

# 安全与审计快速清单（面试/上线前必查）

- 访问控制是否严格（onlyOwner/roles）
    
- 重入（reentrancy）保护（use checks-effects-interactions + ReentrancyGuard）
    
- 溢出/下溢（Solidity >=0.8 默认检查）
    
- 敏感函数是否有合理限制（时间、速率限制）
    
- 外部调用是否安全（慎用 delegatecall）
    
- 随机数来源是否安全（不要用 block.timestamp 做关键随机）
    
- 事件是否记录关键操作（便于链上审计）
    
- Gas 消耗是否可控（避免循环中写 storage）
    
- 测试覆盖率 + 模糊测试（Echidna） + 静态分析（Slither）
